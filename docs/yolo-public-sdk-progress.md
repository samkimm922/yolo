## 自动执行规则

按下面 Todo 从上往下执行。每个 Todo 必须满足：

- 修改前先读相关代码、测试和文档。
- 每个模块化改动必须有对应单测或源代码锚点测试。
- 每批至少跑目标测试；触及 runner/SDK/export 时跑全量测试和 `node --check`。
- 完成后更新本文状态和 `docs/sdk-gap-matrix.md` 的批次记录。
- 测试产生的临时 evidence 要清理。

自动停止并沟通的情况：

- 需要改变 public API、package export、CLI 参数或数据 schema 的兼容承诺。
- 测试失败但根因不明确，或者修复需要扩大到非当前 Todo 的核心边界。
- 发现现有行为存在高风险逻辑 bug，继续抽模块会固化错误行为。
- 需要删除/迁移大量根目录入口或 legacy 文件。
